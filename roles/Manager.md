# 项目经理 (Project Manager)

## 职责
- 整体项目进度把控
- 协调各角色工作
- 风险评估与问题解决
- 确保项目按时交付

## 项目进度管理

### 阶段一：需求与设计 (第 1-2 天)
- [ ] 产品经理输出需求文档
- [ ] 设计师完成 UI 设计稿
- [ ] 技术方案评审

### 阶段二：开发实施 (第 3-5 天)
- [ ] 前端完成页面开发
- [ ] 后端完成云函数开发
- [ ] 数据库权限配置

### 阶段三：测试验收 (第 6-7 天)
- [ ] 测试工程师完成功能测试
- [ ] Bug 修复
- [ ] 产品验收

### 阶段四：上线部署 (第 8 天)
- [ ] 提交审核
- [ ] 发布上线

## 每日站会
**时间：** 每天 10:00
**内容：**
- 昨天完成了什么
- 今天计划做什么
- 遇到什么问题

## 风险管理
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 开发者工具版本不兼容 | 高 | 提前测试环境 |
| 云函数调用超时 | 中 | 优化查询逻辑 |
| 图片上传失败 | 高 | 增加重试机制 |
| 审核不通过 | 中 | 提前了解审核规则 |

## 沟通机制
- 问题及时反馈
- 每日进度同步
- 阻塞问题优先解决
