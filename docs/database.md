# 数据库设计文档 - 校园小猫论坛

> 使用微信云开发（CloudBase）提供的 MongoDB 数据库。

---

## 一、集合总览

| 集合名     | 说明         | 主要用途               |
| ---------- | ------------ | ---------------------- |
| `cats`     | 猫咪信息集合 | 存储猫咪帖子           |
| `users`    | 用户信息集合 | 存储用户档案与收藏记录 |
| `comments` | 评论集合     | 存储评论数据           |

---

## 二、集合详细设计

### 2.1 cats 集合（猫咪信息）

| 字段名        | 类型      | 必填 | 默认值    | 说明                        |
| ------------- | --------- | ---- | --------- | --------------------------- |
| `_id`         | String    | 是   | 自动生成  | 文档唯一标识                |
| `name`        | String    | 是   | -         | 猫咪名字                    |
| `location`    | String    | 是   | -         | 猫咪出现地点（如"三食堂门口"）|
| `description` | String    | 否   | `""`      | 详细描述                    |
| `images`      | String[]  | 是   | `[]`      | 图片云存储 fileID 数组      |
| `tags`        | String[]  | 否   | `[]`      | 标签数组（如"橘猫""亲人"）   |
| `createdAt`   | Date      | 是   | 当前时间  | 创建时间                    |
| `updatedAt`   | Date      | 是   | 当前时间  | 最后更新时间                |
| `createdBy`   | String    | 是   | -         | 发布者 openid               |
| `likes`       | Number    | 是   | `0`       | 点赞数                      |
| `viewCount`   | Number    | 是   | `0`       | 浏览次数                    |
| `status`      | String    | 是   | `"active"`| 状态：`active` / `deleted`  |

### 2.2 users 集合（用户信息）

| 字段名      | 类型      | 必填 | 默认值    | 说明                     |
| ----------- | --------- | ---- | --------- | ------------------------ |
| `_id`       | String    | 是   | 自动生成  | 文档唯一标识             |
| `openid`    | String    | 是   | -         | 微信用户唯一标识         |
| `nickname`  | String    | 是   | `"匿名用户"` | 昵称                   |
| `avatar`    | String    | 否   | `""`      | 头像云存储 fileID        |
| `favorites` | String[]  | 是   | `[]`      | 收藏的猫咪 _id 数组      |
| `createdAt` | Date      | 是   | 当前时间  | 注册时间                 |
| `updatedAt` | Date      | 是   | 当前时间  | 最后更新时间             |

### 2.3 comments 集合（评论）

| 字段名      | 类型      | 必填 | 默认值    | 说明                     |
| ----------- | --------- | ---- | --------- | ------------------------ |
| `_id`       | String    | 是   | 自动生成  | 文档唯一标识             |
| `catId`     | String    | 是   | -         | 关联猫咪的 _id           |
| `userId`    | String    | 是   | -         | 评论者 openid            |
| `nickname`  | String    | 是   | -         | 评论者昵称（冗余存储）   |
| `avatar`    | String    | 否   | `""`      | 评论者头像               |
| `content`   | String    | 是   | -         | 评论内容                 |
| `createdAt` | Date      | 是   | 当前时间  | 评论时间                 |
| `status`    | String    | 是   | `"active"`| 状态：`active` / `deleted`|

## 三、索引设计

### 3.1 cats 集合索引

| 索引字段              | 类型   | 用途                               |
| --------------------- | ------ | ---------------------------------- |
| `createdBy`           | 普通   | 查询某用户发布的所有帖子           |
| `createdAt`           | 普通   | 按时间排序（最新优先）             |
| `tags`                | 数组   | 按标签筛选猫咪                     |
| `location`            | 普通   | 按地点搜索                         |
| `status` + `createdAt`| 复合   | 高效查询有效帖子并按时间排序       |

### 3.2 users 集合索引

| 索引字段 | 类型   | 用途                     |
| -------- | ------ | ------------------------ |
| `openid` | 唯一   | 快速查找/去重用户        |

### 3.3 comments 集合索引

| 索引字段              | 类型   | 用途                               |
| --------------------- | ------ | ---------------------------------- |
| `catId`               | 普通   | 查询某猫咪的所有评论               |
| `catId` + `createdAt` | 复合   | 查询某猫咪评论并按时间排序         |
| `userId`              | 普通   | 查询某用户的所有评论               |

## 四、集合初始化说明

### 4.1 在微信云开发控制台创建集合

1. 登录微信开发者工具 → 点击「云开发」按钮
2. 进入「数据库」页面
3. 点击「+」创建以下三个集合：
   - `cats`
   - `users`
   - `comments`

### 4.2 权限配置

在云开发控制台 → 数据库 → 集合设置中，配置如下权限：

| 集合     | 权限模式   | 规则说明                                     |
| -------- | ---------- | -------------------------------------------- |
| `cats`   | 自定义权限 | 所有用户可读；仅创建者可写、删                |
| `users`  | 自定义权限 | 仅用户本人可读写自己的数据                    |
| `comments`| 自定义权限| 所有用户可读；仅创建者可写、删                |

### 4.3 安全规则示例（cats 集合）

```json
{
  "read": true,
  "write": "auth.openid == doc.createdBy",
  "create": "auth.openid != null"
}
```

### 4.4 初始化脚本（可选）

如需批量导入测试数据，可在云开发控制台 → 数据库 → 集合中点击「导入」，使用以下 JSON 格式：

```json
[
  {
    "name": "大黄",
    "location": "三食堂门口",
    "description": "一只胖乎乎的橘猫，特别亲人",
    "images": [],
    "tags": ["橘猫", "亲人", "胖"],
    "createdBy": "test-openid-001",
    "likes": 0,
    "viewCount": 0,
    "status": "active",
    "createdAt": {"$date": "2026-04-19T10:00:00Z"},
    "updatedAt": {"$date": "2026-04-19T10:00:00Z"}
  }
]
```
