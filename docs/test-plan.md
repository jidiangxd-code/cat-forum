# 校园小猫论坛 - 测试计划

> **项目名称：** 山西机电小猫之家（微信小程序）
> **编写日期：** 2026-04-19
> **版本：** v1.0
> **编写人：** QA Engineer

---

## 一、测试目标

确保「山西机电小猫之家」微信小程序的核心功能（帖子浏览、发布、详情查看）在目标平台和设备上正常运行，达到上线发布标准。

---

## 二、测试阶段划分

### 阶段一：冒烟测试（Smoke Testing）

| 项目 | 说明 |
|------|------|
| **时机** | 每次版本构建后，进入正式测试前 |
| **目标** | 验证核心链路是否通畅 |
| **范围** | 首页加载 → 发布帖子 → 查看详情 → 返回首页 |
| **通过标准** | 核心链路无阻塞性 Bug |
| **预估耗时** | 30 分钟 |

**冒烟用例（必须全部通过）：**
- TC-001 首页帖子列表展示
- TC-006 发布页面正常打开
- TC-014 填写完整信息后成功发布
- TC-017 通过首页点击进入详情页
- TC-022 TabBar 切换到首页
- TC-023 TabBar 切换到发布页

### 阶段二：功能测试（Functional Testing）

| 项目 | 说明 |
|------|------|
| **时机** | 冒烟测试通过后 |
| **目标** | 按测试用例逐项验证所有功能 |
| **范围** | 全部功能测试用例（TC-001 ~ TC-024） |
| **通过标准** | P0 用例 100% 通过，P1 用例通过率 ≥ 90% |
| **预估耗时** | 2-3 小时 |

### 阶段三：兼容性与性能测试

| 项目 | 说明 |
|------|------|
| **时机** | 功能测试通过后 |
| **目标** | 验证多设备、多版本兼容性，以及性能指标 |
| **范围** | 兼容性测试用例（TC-025 ~ TC-039）、性能测试用例（TC-040 ~ TC-049） |
| **通过标准** | P0 设备全部通过，核心性能指标达标 |
| **预估耗时** | 3-4 小时 |

### 阶段四：边界与异常测试

| 项目 | 说明 |
|------|------|
| **时机** | 兼容性与性能测试通过后 |
| **目标** | 验证边界条件下的稳定性和健壮性 |
| **范围** | 边界测试用例（TC-050 ~ TC-072） |
| **通过标准** | 不出现崩溃、数据丢失、安全漏洞 |
| **预估耗时** | 2 小时 |

### 阶段五：回归测试（Regression Testing）

| 项目 | 说明 |
|------|------|
| **时机** | Bug 修复后，新版本发布前 |
| **目标** | 验证已修复 Bug 不再出现，且未引入新问题 |
| **范围** | 与修复 Bug 相关的用例 + 冒烟测试用例 |
| **通过标准** | 修复的 Bug 不再复现，冒烟用例 100% 通过 |
| **预估耗时** | 1-2 小时 |

### 阶段六：验收测试（Acceptance Testing）

| 项目 | 说明 |
|------|------|
| **时机** | 回归测试通过后，发布前 |
| **目标** | 产品方最终确认 |
| **范围** | 全量核心功能走查 |
| **通过标准** | 产品方确认可以发布 |
| **预估耗时** | 1 小时 |

## 三、测试工具和方法

### 3.1 测试工具

| 工具 | 用途 | 说明 |
|------|------|------|
| **微信开发者工具** | 主要测试环境 | 模拟运行、调试、性能分析 |
| **微信真机调试** | 真机验证 | 连接真实手机进行功能验证 |
| **微信体验版** | 团队共享测试 | 上传体验版供多设备测试 |
| **云开发控制台** | 数据验证 | 检查数据库、存储数据 |
| **AppData 面板** | 数据状态检查 | 在开发者工具中查看页面 data 状态 |
| **Network 面板** | 网络请求监控 | 检查网络请求和响应 |
| **Performance 面板** | 性能分析 | 分析页面渲染和 JS 执行性能 |

### 3.2 测试方法

| 方法 | 说明 |
|------|------|
| **黑盒测试** | 不关注代码实现，从用户角度验证功能 |
| **探索性测试** | 在无明确用例的场景下自由探索，发现隐藏 Bug |
| **对比测试** | 同一操作在不同设备/版本上对比结果 |
| **Monkey 测试** | 随机点击、滑动、输入，验证稳定性 |
| **中断测试** | 来电、锁屏、切后台、杀进程等中断场景 |

### 3.3 测试环境配置

| 环境 | 配置 |
|------|------|
| 开发环境 | 微信开发者工具（最新稳定版） |
| 测试环境 | 微信体验版 |
| 测试数据 | 使用本地 globalData 模拟数据 |
| 网络环境 | WiFi / 4G / 弱网（开发者工具 Network 面板模拟） |

## 四、Bug 等级定义

### 4.1 等级分类

| 等级 | 名称 | 定义 | 响应时间 | 修复时限 |
|------|------|------|----------|----------|
| **P0** | 致命（Blocker） | 系统崩溃、数据丢失、核心功能完全不可用 | 立即 | 4 小时内 |
| **P1** | 严重（Critical） | 核心功能异常但有绕过方案，严重影响用户体验 | 2 小时内 | 24 小时内 |
| **P2** | 一般（Major） | 非核心功能异常，UI 错位，体验问题 | 1 个工作日内 | 当前迭代内 |
| **P3** | 轻微（Minor） | 文案错误、颜色偏差、微小 UI 瑕疵 | 排期处理 | 下个迭代 |

### 4.2 Bug 生命周期

```
新建 (New) → 确认 (Confirmed) → 修复中 (Fixing) → 已修复 (Fixed) → 验证通过 (Verified/Closed)
                                              ↓
                                        重新打开 (Reopened) → 修复中
```

### 4.3 Bug 状态说明

| 状态 | 说明 |
|------|------|
| **New** | 新发现的 Bug，尚未确认 |
| **Confirmed** | 已确认是 Bug，分配给开发人员 |
| **Fixing** | 开发人员正在修复 |
| **Fixed** | 开发人员已修复，待验证 |
| **Verified** | QA 验证通过，关闭 |
| **Reopened** | 验证未通过，重新打开 |
| **Won't Fix** | 确认不修复（需产品确认） |
| **Duplicate** | 重复 Bug |

## 六、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据存储仅用 globalData，杀进程后数据丢失 | 用户体验差 | 当前阶段可接受，后续迭代接入云数据库 |
| 图片未上传云端，仅用本地临时路径 | 他人无法查看图片 | 后续接入云存储，当前阶段可接受 |
| 无内容安全审核 | 可能发布违规内容 | 接入微信内容安全 API（云函数校验） |
| 真机设备有限 | 兼容性覆盖不足 | 优先测试 iOS + Android 各一款主力机型 |
| 无自动化测试 | 回归测试效率低 | 当前手动测试为主，后续考虑引入自动化框架 |
