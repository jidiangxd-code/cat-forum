# 校园小猫论坛 - 需求文档

## 1. 产品概述

校园小猫论坛是一款基于微信小程序的社区平台，致力于为高校师生及爱猫人士提供一个交流、分享校园猫咪信息的空间。用户可以通过本平台发布猫咪照片、记录猫咪动态、查询猫咪档案，共同关注和保护校园猫咪的日常生活。

### 1.1 产品定位

- **产品形态**：微信小程序（免安装、即用即走）
- **核心场景**：发现校园猫咪 → 分享猫咪动态 → 查看猫咪档案 → 社区互动
- **差异化优势**：聚焦单一校园场景，形成高粘性的垂直社区

### 1.2 产品目标

| 目标维度 | 具体指标 |
|---------|---------|
| 用户增长 | 上线 3 个月内覆盖目标校园 60% 以上师生 |
| 内容沉淀 | 建立校园猫咪档案库，覆盖校园内 90% 以上的常驻猫咪 |
| 社区活跃 | 日均发帖量 > 50 条，日活用户 > 500 人 |
| 用户留存 | 7 日留存率 > 30%，30 日留存率 > 15% |

## 2. 目标用户

### 2.1 核心用户画像

| 用户角色 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 爱猫学生 | 在校大学生/研究生，喜欢猫咪，经常在校园遇到猫 | 分享猫咪日常、了解猫咪信息、找到同好 |
| 校园猫咪关注者 | 长期关注校园猫咪，可能有投喂行为 | 记录猫咪状态、追踪猫咪健康、提醒注意事项 |
| 校园管理员/后勤 | 关心校园猫咪的管理与安全 | 了解猫咪数量、健康状况、绝育进度 |
| 潜在用户 | 刚入学的新生、教职工 | 了解校园猫咪文化、融入社区 |

### 2.2 用户场景

- **场景一**：在校园散步时偶遇一只陌生猫咪，打开小程序拍照发布并询问"这是谁？"
- **场景二**：想知道某只常出没的猫咪最近状况如何，搜索并查看它的档案
- **场景三**：看到有人发帖说猫咪受伤，前往查看并提供帮助
- **场景四**：浏览首页，发现校园猫咪的最新趣闻和照片

## 3. 功能需求

### 3.1 首页模块

首页是用户打开小程序后看到的第一屏，承担内容分发和用户引导的功能。

#### 3.1.1 功能清单

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 热门帖子流 | 按时间/热度排序展示帖子，支持下拉刷新、上拉加载更多 | P0 |
| 猫咪卡片推荐 | 每日推荐 1-3 只校园明星猫咪，展示基本信息 | P0 |
| 搜索入口 | 支持搜索猫咪名称、标签、帖子内容 | P0 |
| 公告/通知栏 | 顶部滚动展示社区公告、猫咪救助信息等 | P1 |
| 分类标签筛选 | 按"日常""救助""绝育""领养"等标签过滤内容 | P1 |
| 定位切换 | 支持切换不同校区（适用于多校区学校） | P2 |

#### 3.1.2 交互说明

- 帖子列表采用瀑布流布局，每张帖子展示封面图、标题/摘要、发布者、点赞数
- 点击帖子进入详情页
- 下拉刷新时显示加载动画，刷新后重新拉取数据
- 上拉触底时自动加载下一页

### 3.2 发现模块

发现模块是用户探索校园猫咪的核心入口。

#### 3.2.1 猫咪档案库

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 猫咪列表 | 以卡片形式展示所有已录入的校园猫咪 | P0 |
| 猫咪详情页 | 展示猫咪照片、昵称、花色、性别（预估）、出没地点、健康状态、绝育状态、登记时间 | P0 |
| 猫咪动态时间线 | 按时间倒序展示该猫咪的所有相关帖子 | P0 |
| 猫咪标签 | "已绝育""已驱虫""怀孕中""需要关注"等健康标签 | P0 |
| 认领/关注猫咪 | 用户可关注特定猫咪，有新动态时收到提醒 | P1 |

#### 3.2.2 猫咪地图

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 校园地图展示 | 在校园地图上标注猫咪出没热点 | P2 |
| 出没记录 | 点击标注点查看该地点的猫咪出没记录 | P2 |

#### 3.2.3 排行榜

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 最受欢迎猫咪 | 按关注数/帖子数排序 | P1 |
| 活跃用户排行 | 按发帖数/互动数排序 | P2 |

### 3.3 发布模块

发布模块是社区内容生产的入口。

#### 3.3.1 发帖功能

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 图片上传 | 支持 1-9 张图片上传，可编辑顺序，支持预览和删除 | P0 |
| 文字输入 | 支持 500 字以内的文字描述 | P0 |
| 关联猫咪 | 发帖时可关联已有猫咪档案或创建新猫咪档案 | P0 |
| 位置标记 | 自动获取当前位置，可选择校园内的地点标签 | P0 |
| 分类标签 | 选择帖子类型：日常/救助/绝育/领养/寻猫/其他 | P0 |
| @提醒 | 支持 @ 其他用户（可选） | P2 |

#### 3.3.2 发布流程

```
选择图片 → 填写描述 → 选择分类标签 → 关联猫咪(可选) → 标记位置 → 预览 → 发布
```

### 3.4 个人中心模块

#### 3.4.1 个人资料

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 基本信息 | 头像（微信授权）、昵称、个人简介 | P0 |
| 我的帖子 | 展示自己发布的所有帖子 | P0 |
| 我的收藏 | 收藏的帖子/猫咪 | P1 |
| 关注/粉丝 | 关注列表和粉丝列表 | P1 |
| 我的猫咪 | 我创建/管理的猫咪档案 | P0 |

#### 3.4.2 消息通知

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 点赞通知 | 有人点赞我的帖子时推送 | P0 |
| 评论通知 | 有人评论我的帖子时推送 | P0 |
| 关注通知 | 有人关注我时推送 | P1 |
| 猫咪动态通知 | 关注的猫咪有新动态时推送 | P1 |
| 系统通知 | 管理员发布的系统消息 | P1 |

#### 3.4.3 设置

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 通知开关 | 控制各类通知的推送开关 | P1 |
| 隐私设置 | 控制个人资料的可见范围 | P2 |
| 关于我们 | 产品介绍、使用指南、反馈入口 | P1 |
| 清理缓存 | 清除本地缓存数据 | P2 |

### 3.5 帖子详情与互动（补充模块）

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 帖子详情 | 展示图片轮播、文字描述、发布时间、发布者信息、位置 | P0 |
| 点赞 | 一键点赞，显示点赞数 | P0 |
| 评论 | 支持文字评论，支持回复子评论（最多两层） | P0 |
| 分享 | 分享至微信好友/朋友圈 | P0 |
| 举报 | 举报不当内容 | P1 |

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|-----|------|
| 首屏加载时间 | ≤ 2 秒（4G 网络） |
| 图片加载 | 缩略图 ≤ 1 秒，原图 ≤ 3 秒 |
| API 响应时间 | 常规接口 ≤ 500ms |
| 并发支持 | 支持 1000 用户同时在线 |

### 4.2 安全要求

| 要求 | 说明 |
|-----|------|
| 用户认证 | 基于微信开放平台登录，获取用户唯一标识 |
| 内容安全 | 接入微信内容安全 API，自动过滤敏感/违规图片和文字 |
| 数据保护 | 用户隐私数据加密存储，不泄露个人信息 |
| 权限控制 | 仅帖子作者本人可编辑/删除自己的帖子 |

### 4.3 兼容性要求

| 要求 | 说明 |
|-----|------|
| 微信版本 | 支持微信 7.0 及以上版本 |
| 手机系统 | iOS 12+ / Android 8.0+ |
| 屏幕适配 | 适配主流手机屏幕尺寸（iPhone SE 至 Max，Android 主流机型） |
| 深色模式 | 支持跟随系统切换深色/浅色模式 |

### 4.4 可用性要求

| 要求 | 说明 |
|-----|------|
| 服务可用性 | ≥ 99.5% |
| 离线能力 | 已加载的帖子在离线状态下可浏览 |
| 容错处理 | 网络异常时展示友好提示，支持重试 |

### 4.5 数据与存储

| 要求 | 说明 |
|-----|------|
| 图片存储 | 单张图片 ≤ 5MB，自动压缩优化 |
| 数据库 | 使用云数据库，支持数据备份与恢复 |
| 数据保留 | 用户删除帖子后保留 30 天可恢复 |

## 5. 页面流程

### 5.1 整体页面架构

```
┌─────────────────────────────────────────┐
│            TabBar (底部导航)              │
│   [首页]     [发现]     [发布]     [我的]  │
└─────────────────────────────────────────┘
```

### 5.2 主要页面及跳转关系

```
首页
 ├── 帖子详情 ── 评论列表
 │              ── 用户主页
 │              ── 猫咪详情
 ├── 猫咪推荐卡片 ── 猫咪详情
 ├── 搜索 ── 搜索结果（帖子/猫咪）
 │
发现
 ├── 猫咪列表 ── 猫咪详情 ── 猫咪动态时间线
 ├── 猫咪地图
 ├── 排行榜
 │
发布
 ├── 拍照/选图
 ├── 编辑帖子（文字+标签+位置+关联猫咪）
 ├── 预览
 ├── 发布成功 ── 返回上一页 / 查看帖子
 │
我的
 ├── 个人资料 ── 编辑资料
 ├── 我的帖子 ── 帖子详情 / 编辑 / 删除
 ├── 我的收藏 ── 帖子详情
 ├── 关注/粉丝 ── 用户主页
 ├── 我的猫咪 ── 猫咪详情 / 编辑猫咪档案
 ├── 消息通知 ── 点击跳转对应内容
 ├── 设置
```

### 5.3 关键用户流程

#### 流程一：首次使用

```
打开小程序 → 微信授权登录 → 选择所在校区 → 进入首页（引导页，可选）
```

#### 流程二：发布帖子

```
点击底部"+" → 选择图片 → 填写描述 → 选择分类 → 关联猫咪(可选) → 标记位置 → 确认发布
```

#### 流程三：创建猫咪档案

```
发现页 → 点击"新增猫咪" → 上传照片 → 填写昵称/花色/出没地 → 提交审核 → 档案创建成功
```

#### 流程四：查看猫咪动态

```
发现页 → 点击猫咪卡片 → 进入猫咪详情 → 浏览动态时间线 → 点击帖子查看详情
```

## 6. 功能模块清单（V1.0 MVP）

以下为 V1.0 最小可行产品（MVP）的功能模块优先级划分：

### P0 - 必须上线（MVP 核心）

| 模块编号 | 模块名称 | 功能点 |
|---------|---------|-------|
| M01 | 用户系统 | 微信授权登录、获取用户信息 |
| M02 | 首页信息流 | 帖子列表、下拉刷新、上拉加载 |
| M03 | 帖子发布 | 图片上传(1-9张)、文字描述、分类标签、位置标记 |
| M04 | 帖子详情 | 图片轮播、详情展示、点赞、评论、分享 |
| M05 | 猫咪档案 | 猫咪列表、猫咪详情、猫咪动态关联 |
| M06 | 个人中心 | 我的帖子、我的猫咪、基本信息展示 |

### P1 - 重要但不阻塞上线

| 模块编号 | 模块名称 | 功能点 |
|---------|---------|-------|
| M07 | 消息通知 | 点赞/评论/关注/猫咪动态通知 |
| M08 | 搜索功能 | 帖子搜索、猫咪搜索 |
| M09 | 收藏功能 | 收藏帖子、收藏猫咪 |
| M10 | 关注系统 | 关注用户、粉丝列表 |
| M11 | 内容安全 | 图片/文字自动审核 |
| M12 | 分类筛选 | 按标签过滤帖子 |

### P2 - 锦上添花（后续迭代）

| 模块编号 | 模块名称 | 功能点 |
|---------|---------|-------|
| M13 | 猫咪地图 | 校园地图标注猫咪出没点 |
| M14 | 排行榜 | 最受欢迎猫咪、活跃用户排行 |
| M15 | 深色模式 | 跟随系统的深色模式适配 |
| M16 | 多校区支持 | 校区切换 |
