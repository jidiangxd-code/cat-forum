> **审查人：** 产品经理

> **输入文档：** product-review-v2.md / requirements.md / user-stories.md

---

## 一、执行摘要

基于第一轮产品审查（product-review-v2.md）的结论，当前 MVP 完成度约 **55%**。本文档将所有缺失和待完善功能按 **P0 阻塞 → P1 重要 → P2 优化** 三级优先级排序，为开发团队提供明确的行动清单。

**核心判断：** 当前状态无法通过微信审核。必须优先修复 5 个阻塞性问题，再逐步完善体验。

---

## 二、P0 紧急 — 阻塞 MVP 上线（预计 3-5 个工作日）

以下功能缺失将导致：① 小程序审核被拒 ② 核心用户流程断裂 ③ 严重体验事故。

### P0-1 🔴 内容安全审核（最高优先级）

| 维度 | 详情 |
|------|------|
| **问题** | UGC 社区未接入微信内容安全 API，审核 **100% 被拒** |
| **涉及需求** | M11、US 非功能需求 4.2 安全要求 |
| **缺失范围** | `security.msgSecCheck`（文字审核）、`security.imgSecCheck`（图片审核）均未接入 |
| **实现方案** | 创建云函数 `checkContent`，统一处理文字+图片审核：<br>1. 用户提交 → 云函数接收<br>2. 先调 `imgSecCheck`（图片上传到云存储后）<br>3. 再调 `msgSecCheck`（帖子文字/评论文字）<br>4. 审核通过 → 写入数据库；不通过 → 返回错误提示<br>5. 发布页和评论页的提交逻辑改为调用此云函数 |
| **验收标准** | - 发布含敏感词的帖子时，提示"内容包含违规信息，请修改后重试"<br>- 上传违规图片时，提示"图片审核未通过"<br>- 正常内容发布流程不受影响（增加 <1 秒延迟） |
| **工时估算** | 1 天 |
| **负责人** | 后端 + 前端 |

### P0-2 🔴 微信授权登录

| 维度 | 详情 |
|------|------|
| **问题** | 无微信登录，所有用户身份为 "guest"，发布署名为"我" |
| **涉及需求** | M01、US-001 |
| **缺失范围** | `wx.login()`、`wx.getUserProfile()` 均未调用，无授权引导页 |
| **实现方案** | 1. `app.js` 的 `onLaunch` 中检查登录态<br>2. 无登录态 → 展示授权引导页（非弹窗阻塞式，允许先浏览）<br>3. 用户点击"微信登录" → `wx.login()` 获取 code → 云函数换取 openid<br>4. 存储 token，二次打开自动登录<br>5. 拒绝授权时进入「只读模式」（可浏览，不可发布/评论） |
| **验收标准** | - 首次打开弹出/引导微信授权<br>- 授权后自动获取头像和昵称<br>- 拒绝授权时可浏览但不可发帖<br>- 二次打开自动登录 |
| **工时估算** | 1.5 天 |
| **负责人** | 前端 + 后端 |

| 维度 | 详情 |

| **问题** | 用户无法搜索猫咪或帖子，`api.js` 有 `searchCats` 但无搜索页面 |
| **涉及需求** | M08、US-012 |
| **实现方案** | 1. 创建 `pages/search/search` 页面<br>2. 首页顶部增加搜索框入口（点击跳转搜索页）<br>3. 搜索结果分两个 Tab：「猫咪」和「帖子」<br>4. 搜索历史记录使用 `wx.setStorageSync` 本地存储（最近 10 条）<br>5. 支持按猫咪昵称、帖子描述关键词模糊搜索 |
| **验收标准** | - 首页可进入搜索页<br>- 支持猫咪昵称搜索<br>- 支持帖子内容关键词搜索<br>- 搜索结果分"猫咪"和"帖子"两个 Tab<br>- 展示搜索历史记录 |
| **工时估算** | 2 天 |

| 维度 | 详情 |

| **涉及需求** | M07、US-021 |

| **验收标准** | - "我的"页面显示未读消息数量角标<br>- 消息列表分类型展示（点赞/评论/关注/系统）<br>- 点击消息可跳转到对应帖子/评论<br>- 支持全部已读操作<br>- 接入微信订阅消息推送 |

### P1-4 🟡 子评论（回复功能）

| 维度 | 详情 |
