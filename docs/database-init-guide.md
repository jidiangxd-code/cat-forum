# 数据库初始化指南（统一方案）

> **更新时间:** 2026-05-05 22:10  
> **目标:** 统一使用云函数初始化数据库，避免手动创建遗漏

---

## 一、为什么统一使用云函数？

### 对比分析

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **云函数创建** | ✅ 可复现<br>✅ 版本控制<br>✅ 不易遗漏<br>✅ 一键部署 | ⚠️ 权限需手动设置 | **生产环境**<br>多环境部署<br>团队协作 |
| **手动创建** | ✅ 可视化操作<br>✅ 即时生效 | ❌ 不可复现<br>❌ 易遗漏<br>❌ 无记录 | 快速测试<br>临时集合 |

### 结论

**统一使用云函数创建集合**，原因：
1. **可复现：** 换环境重新运行即可
2. **版本控制：** 代码中有完整记录
3. **不易遗漏：** 列表清晰，不会忘记
4. **团队协作：** 团队成员运行同一个云函数即可

## 二、初始化步骤（3 步）

### 步骤 1：上传云函数

```
在微信开发者工具中：
1. 找到 cloudfunctions/init-collections 文件夹
2. 右键点击 → 选择「上传并部署：云端安装依赖」
3. 等待上传完成（约 10-30 秒）
```

### 步骤 2：运行云函数

```
上传完成后：
1. 右键点击 init-collections 文件夹
2. 选择「云端测试」
3. 点击「调用」按钮
4. 查看返回结果
```

**预期返回结果：**
```json
{
  "success": true,
  "summary": {
    "total": 10,
    "created": 10,        // 或部分已存在
    "alreadyExisted": 0,
    "failed": 0
  },
  "details": [
    {
      "collection": "cats_profile",
      "description": "猫咪档案",
      "status": "✅ created",
      "permission": "所有用户可读写"
    },
    // ... 其他集合
  ],
  "tip": "✅ 集合创建完成！请按照上方指南手动设置权限和索引。"
}
```

### 步骤 3：设置权限（必须手动）

**为什么不能自动设置权限？**
- 云开发 API 不支持通过云函数设置权限
- 权限设置涉及安全策略，必须人工确认

**设置步骤：**
```
1. 打开云开发控制台（点击工具栏「云开发」按钮）
2. 进入「数据库」标签页
3. 对每个集合设置权限：
```

**权限配置表：**

| 集合名称 | 说明 | 权限设置 | 原因 |
|---------|------|---------|------|
| `cats_profile` | 猫咪档案 | **所有用户可读写** | 猫咪档案公开，任何人可创建和编辑 |
| `posts` | 帖子 | **所有用户可读，仅创建者可写** | 帖子公开，仅作者可编辑 |
| `comments` | 评论 | **所有用户可读，仅创建者可写** | 评论公开，仅作者可删除 |
| `votes` | 投票 | **仅创建者可读写** | 投票记录私有 |
| `favorites` | 收藏 | **仅创建者可读写** | 收藏记录私有 |
| `notifications` | 通知 | **仅创建者可读写** | 通知私有 |
| `reports` | 举报 | **仅创建者可读写** | 举报记录私有 |
| `follows` | 关注关系 | **仅创建者可读写** | 关注关系私有 |
| `users` | 用户信息 | **所有用户可读，仅创建者可写** | 用户信息公开，仅自己可编辑 |
| `debug_logs` | 错误日志 | **仅创建者可读写** | 错误日志私有 |

## 三、索引设置（可选，提升性能）

### 为什么需要索引？

**没有索引：**
```
查询 posts 集合中 catId = 'xxx' 的帖子
→ 全表扫描（慢）
```

**有索引：**
```
查询 posts 集合中 catId = 'xxx' 的帖子
→ 直接定位（快）
```

### 索引配置表

| 集合 | 索引字段 | 索引类型 | 用途 |
|------|---------|---------|------|
| `users` | `openid` | 唯一索引 | 快速查找用户 |
| `posts` | `catId` | 普通索引 | 查询某只猫的帖子 |
| `posts` | `createTime` | 普通索引 | 按时间排序 |
| `comments` | `postId` | 普通索引 | 查询某帖子的评论 |
| `comments` | `createTime` | 普通索引 | 按时间排序 |
| `cats_profile` | `catType` | 普通索引 | 按类型筛选 |
| `cats_profile` | `status` | 普通索引 | 按状态筛选 |
| `favorites` | `userOpenid` | 普通索引 | 查询用户收藏 |
| `notifications` | `toOpenid` | 普通索引 | 查询用户通知 |

### 设置步骤

```
云开发控制台 → 数据库 → 选择集合 → 索引管理 → 添加索引
```

## 五、常见问题

### Q1: 云函数运行失败怎么办？

**可能原因：**
- 云环境未初始化
- 云函数未上传
- 网络问题

**解决方案：**
```
1. 确认云开发环境已创建（云开发控制台 → 设置 → 环境ID）
2. 重新上传云函数（右键 → 上传并部署）
3. 检查网络连接
```

### Q2: 集合已存在怎么办？

**不用担心：**
- 云函数会自动检测集合是否存在
- 已存在的集合会跳过创建
- 返回结果中会显示 `already_exists`

### Q3: 权限设置错误会怎样？

**可能问题：**
- 权限过严：功能无法使用（如无法发帖）
- 权限过宽：数据安全问题（如他人可修改你的帖子）

**解决方案：**
- 按照上方权限配置表设置
- 不确定时选择更严格的权限

### Q4: 需要添加新集合怎么办？

**步骤：**
```
1. 编辑 cloudfunctions/init-collections/index.js
2. 在 collections 数组中添加新集合配置
3. 重新上传并运行云函数
4. 手动设置权限
```

**示例：**
```javascript
const collections = [
  // ... 现有集合
  {
    name: 'new_collection',
    description: '新集合',
    permission: '所有用户可读，仅创建者可写',
    indexes: ['field1', 'field2']
  }
]
```

## 六、迁移指南（从手动创建迁移）

### 如果你之前手动创建过集合

**情况一：集合已存在**
```
✅ 无需删除
✅ 运行云函数会自动跳过
✅ 只需确认权限设置正确
```

**情况二：集合不存在**
```
✅ 直接运行云函数创建
✅ 然后设置权限
```

**情况三：集合名称不一致**
```
⚠️ 需要手动迁移数据
⚠️ 或修改代码中的集合名称
```
