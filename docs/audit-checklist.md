# 微信小程序审核检查清单

> **编写日期：** 2026-04-19
> **版本：** v1.0

## 一、微信小程序审核常见拒绝原因

### 1.1 功能相关

| 编号 | 拒绝原因 | 说明 | 本项目关联 |
|------|----------|------|------------|
| R-01 | 功能不完整 | 小程序提交审核后，核心功能不可用或未完成 | 发布、浏览功能必须正常 |
| R-02 | 页面无法访问 | 存在空白页、404 页面或死链接 | 三个页面（首页、发布、详情）必须正常 |
| R-03 | 缺少必要交互反馈 | 用户操作后无任何提示 | 发布成功/失败均有 Toast 提示 |
| R-04 | 存在测试内容 | 提交版本中包含明显的测试数据 | 提交前清理测试数据 |

### 1.2 内容安全相关

| 编号 | 拒绝原因 | 说明 | 本项目关联 |
|------|----------|------|------------|
| R-05 | 无内容安全审核 | 用户可发布内容但未做审核 | ⚠️ 当前未接入，需后续接入微信内容安全 API |
| R-06 | 含违法违规内容 | 用户发布的内容包含违法、色情、暴力等信息 | 需接入内容安全校验 |
| R-07 | 包含诱导分享内容 | 强制分享才能使用功能 | 当前无分享功能，不涉及 |

### 1.3 用户体验相关

| 编号 | 拒绝原因 | 说明 | 本项目关联 |
|------|----------|------|------------|
| R-08 | 界面不符合描述 | 小程序实际功能与提交审核时的描述不符 | 描述需与功能一致 |
| R-09 | 交互体验差 | 按钮太小、文字重叠、排版混乱 | 需测试多种机型 |
| R-10 | 加载时间过长 | 首屏加载超过合理时间 | 控制包体积，优化加载 |

### 1.4 隐私与权限相关

| 编号 | 拒绝原因 | 说明 | 本项目关联 |
|------|----------|------|------------|
| R-11 | 滥用用户授权 | 申请了与功能无关的权限 | 当前仅使用 chooseMedia（相册权限） |
| R-12 | 缺少隐私协议 | 收集用户信息但未说明用途 | 用户输入昵称，属于自愿填写 |
| R-13 | 未提供注销/删除 | 收集用户数据但无删除方式 | 当前数据存于 globalData，杀进程即清除 |
| R-14 | 相册权限使用不当 | 使用 chooseMedia 时未合理说明用途 | 选择猫咪照片，用途明确 |

### 1.5 技术规范相关

| 编号 | 拒绝原因 | 说明 | 本项目关联 |
|------|----------|------|------------|
| R-15 | 包体积超限 | 主包超过 2MB 限制 | 需检查主包大小 |
| R-16 | 使用了未开放 API | 调用了未对普通小程序开放的 API | 当前使用的 API 均为公开 API |
| R-17 | 存在 iOS 上架违规 | 包含虚拟支付、iOS 端引导下载等 | 当前不涉及 |
| R-18 | 类目选择错误 | 小程序服务类目与实际功能不符 | 选择合适的类目（如：工具-效率/社交-论坛） |

## 二、自查清单

### 2.1 功能完整性

- [ ] 首页能正常展示帖子列表
- [ ] 发布页面能正常填写信息并发布
- [ ] 详情页能正常展示帖子详情
- [ ] TabBar 切换正常
- [ ] 表单校验正常工作（昵称为空/内容为空/图片未选均提示）
- [ ] 发布成功后有明确反馈（Toast）
- [ ] 发布成功后自动跳转回首页

### 2.2 内容安全

- [ ] 已接入微信内容安全 API（`security.msgSecCheck`）
- [ ] 已接入图片内容安全审核（`security.imgSecCheck`）
- [ ] 违规内容有处理机制（不展示/删除）
- [ ] 有举报或反馈渠道

> ⚠️ **注意：** 用户生成内容（UGC）类小程序必须接入内容安全审核，否则大概率被拒。

### 2.3 用户体验

- [ ] 页面加载速度合理（首屏 < 2 秒）
- [ ] 按钮大小适中（建议 ≥ 44px × 44px）
- [ ] 文字清晰可读，不重叠
- [ ] 图片加载有占位图/加载中的状态
- [ ] 空数据状态有合理提示（当前首页无帖子时无提示，建议添加）
- [ ] 错误状态有合理提示（如图片加载失败）
- [ ] 导航栏标题与实际功能一致

### 2.4 隐私与权限

- [ ] 仅在需要时请求权限（不在启动时立即请求）
- [ ] 权限申请有合理的用途说明
- [ ] 用户拒绝权限后有合理的降级处理
- [ ] 未收集与功能无关的用户信息
- [ ] 隐私政策已配置（小程序后台设置）

### 2.5 技术规范

- [ ] 小程序主包体积 < 2MB
- [ ] 所有页面路径在 app.json 中正确注册
- [ ] 没有未处理的 JS 错误（检查控制台）
- [ ] 没有未处理的 Promise rejection
- [ ] 使用了 wx.chooseMedia 而非已废弃的 wx.chooseImage
- [ ] 基础库版本设置合理（建议 ≥ 2.20.0）
- [ ] sitemap.json 已配置
- [ ] app.json 中 pages 数组顺序正确（首页放第一个）

### 2.6 提交审核前检查

- [ ] 已清理所有 console.log 调试代码
- [ ] 已删除测试数据和模拟数据
- [ ] 小程序简介和描述与实际功能一致
- [ ] 选择了正确的服务类目
- [ ] 上传了合适的截图（建议 3-5 张）
- [ ] 测试账号已准备（如有登录功能）
- [ ] 版本号和版本描述已填写
- [ ] 已在体验版上完整走查一遍

### 2.7 专项类目检查（UGC 用户生成内容类）

由于本项目属于用户生成内容类小程序，以下检查项为**必过项**：

- [ ] **内容安全审核机制已部署**（云函数调用微信安全 API）
- [ ] **有内容管理机制**（删除/隐藏违规内容）
- [ ] **有用户反馈渠道**（举报按钮或客服入口）
- [ ] **隐私政策已填写**（小程序后台 → 设置 → 服务内容声明）
- [ ] **服务类目选择正确**（建议：社交-论坛 或 工具-信息）
- [ ] **小程序名称与功能相关**（不能含"测试""demo"等字样）

## 四、审核被拒应对策略

| 拒绝原因 | 应对策略 |
|----------|----------|
| 功能不完整 | 完善核心功能后重新提交 |
| 无内容审核 | 接入 `security.msgSecCheck` 和 `security.imgSecCheck` |
| 类目不符 | 修改为正确的服务类目 |
| 隐私问题 | 完善隐私政策，减少不必要的权限申请 |
| 体验差 | 优化 UI 和交互后重新提交 |
