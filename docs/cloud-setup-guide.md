# 云开发环境配置指南

## 问题背景

图片存储已从本地临时路径迁移到微信云存储，所有用户可查看同一图片。

**云存储方案：**

- 图片上传到微信云存储，返回 `fileID`
- `fileID` 可直接用于 `<image src="fileID">` 显示
- 云存储自动 CDN 加速，其他用户可访问

## 首次使用配置步骤

### 1. 开通云开发

1. 打开微信开发者工具，打开项目：`C:\Users\郭旭东\Desktop\xiao_mao`
2. 点击工具栏顶部的 **「云开发」** 按钮 ☁️
3. 在弹出窗口中点击 **「开通」**
4. 选择 **「免费基础版」**（个人项目足够使用）
5. 等待环境创建完成

### 2. 获取环境ID

1. 云开发控制台 → **「设置」** → **「环境设置」**
2. 复制 **环境ID**（格式类似 `cloud1-xxxxx`）

### 3. 配置项目

如果自动初始化不生效，可在 `miniprogram/app.js` 中手动指定环境ID：

```javascript
wx.cloud.init({
  env: '你的环境ID',  // 替换为实际环境ID
  traceUser: true
});
```

### 4. 创建数据库集合

在云开发控制台 → **「数据库」** → 创建以下集合：

| 集合名称     | 用途     | 权限                          |
| ------------ | -------- | ----------------------------- |
| `cats`     | 猫咪信息 | 所有用户可读，仅创建者可写/删 |
| `users`    | 用户信息 | 所有用户可读，仅自己可写      |
| `comments` | 评论信息 | 所有用户可读，仅自己可写/删   |

### 5. 设置数据库权限

在云开发控制台 → 每个集合 → **「权限设置」** → 选择 **「自定义权限」**：

**cats 集合：**

```json
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```

**users 集合：**

```json
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```

**comments 集合：**

```json
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```

### 6. 部署云函数（可选）

如果使用云函数方式：

```bash
# 在微信开发者工具中右键每个云函数目录
# 选择「上传并部署：云端安装依赖」
```

需要部署的云函数：

- `cloudfunctions/getCatList/`
- `cloudfunctions/getCatDetail/`
- `cloudfunctions/addCat/`
- `cloudfunctions/deleteCat/`
- `cloudfunctions/addComment/`
- `cloudfunctions/toggleFavorite/`

## 配置验证

完成以上步骤后：

1. 重新编译小程序
2. 进入「发布」页面
3. 上传一张猫咪照片
4. 填写描述后提交
5. 在云开发控制台 → 数据库 → cats 集合，应能看到新记录
6. 在云开发控制台 → 存储，应能看到上传的图片文件

## 存储路径规则

上传的图片按以下规则存储到云存储：

```
云存储根目录/
└── cats/
    ├── 1713520800-abc123-0.jpg
    ├── 1713520801-def456-0.jpg
    └── ...
```

文件名格式：`时间戳-随机字符串-序号.扩展名`

## 图片显示说明

- 发布页：本地图片 → 上传到云存储 → 保存 fileID 到数据库
- 首页/详情页：从数据库读取 fileID → 直接用于 `<image src="fileID">`
- 微信小程序会自动将 fileID 转换为可访问的 CDN URL

## 常见问题

### Q: 图片无法显示？

A: 检查云存储权限设置，确保有读取权限。

### Q: 上传失败？

A: 检查云开发环境是否已开通，环境ID是否正确。

### Q: 云存储容量不够？

A: 免费版有 5GB 存储，可升级到付费版。
