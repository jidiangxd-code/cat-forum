## 三、本次开发任务 —— P1-6 举报功能

### 本次完成：评论通知链路（P1-4 的通知补充）

**问题根因：** `addComment` 云函数原来只写数据库，**从未调用** `sendNotification`。子评论功能（上次完成）虽有 UI 和参数，但没有通知触发。

### 修改文件

#### 1. `cloudfunctions/addComment/index.js` —— 核心改动

```
原问题：
- 只写 comments 集合，不写 notifications
- 用 require('/sendNotification') 云函数内调用报错（路径问题）
- 旧版只支持 catId，新版帖子系统 postId 不兼容

本次改动：
① 重构参数校验：同时支持 postId（新版）和 catId（旧版兼容）
② 评论写成功后，自动写入 notifications：
   - 主评论 → 通知帖子作者
   - 子评论 → 通知被回复用户 + 通知帖子作者（若不同）
③ 评论数 +1 由云函数自动处理（前端乐观更新保留，体验更好）
④ 去掉错误的 require 引用，改用直接 db.collection('notifications').add()
```

#### 2. `miniprogram/utils/api.js` —— addComment 改为调用云函数

```
原来：直接调用 db.collection('comments').add()（不走云函数，无法触发通知）
现在：改为 callCloud('addComment', params)（云函数统一处理写库+通知+计数）
```

#### 3. `miniprogram/pages/detail/detail.js` —— 参数修正

```
- parentId: undefined → ''（空字符串，云函数更好处理）
- 保留前端乐观更新 commentCount（立即可见，体验好）
- 注释更新说明"云函数已自动更新 commentCount"
```

### 通知逻辑设计

```
主评论（无 parentId）：
  → 通知 post.authorId（帖子作者）

子评论（有 parentId）：
  → 通知 replyToUserId（被回复的人）
  → 同时通知 post.authorId（若发帖人不是回复对象，避免重复通知）
```
