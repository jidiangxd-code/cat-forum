## 二、本次开发任务 —— P2-5 GPS 自动定位

### 完成内容

为发布帖子功能新增 GPS 自动定位 + 手动选择校园地点标签功能，用户可选择精确位置或使用预设校园地点。

### 修改文件

#### 1. `miniprogram/pages/publish/publish.js` — **完全重写**

**新增功能：**

- **CAMPUS_LOCATIONS 常量** — 预设 16 个校园地点标签：
  图书馆、食堂门口、宿舍楼下、教学楼、操场、体育馆、行政楼、实验楼、校医院、东门、南门、北门、花园、湖边、自习室、快递站

- **`_autoGetLocation()`** — 页面加载时静默获取 GPS 位置（失败不弹窗，用户无感知）

- **`getGpsLocation()`** — 手动获取 GPS 按钮，含授权引导（未授权时引导授权，已授权直接定位）

- **`_doGetLocation()`** — 执行实际定位，定位成功后自动填充经纬度展示

- **`onLocationTap()`** — 点击地点标签选择/取消，支持单选互斥

- **`clearLocation()`** — 清除已选位置

- **发布时注入位置** — `confirmPublish()` 将 `location`、`latitude`、`longitude` 注入到 `publishPost` 参数

#### 2. `miniprogram/pages/publish/publish.wxml` — **更新**

**新增位置区域（位于「分类」和「绑猫」之间）：**
- GPS 定位按钮（蓝色标签样式，定位中显示加载状态）
- 已定位时显示当前位置标签 + 清除按钮
- 16 个校园地点标签网格（选中态为蓝色）
- 位置区域标记为"可选"

**预览模式增强：**
- 位置信息展示（绿色标签样式）
- 「编辑位置」快捷入口（一键返回编辑页面）

#### 3. `miniprogram/pages/publish/publish.wxss` — **更新**

**新增样式：**
- `.location-top` — GPS按钮+当前定位行布局
- `.gps-btn` — GPS定位按钮样式（蓝色，标签式）
- `.location-current` — 已定位信息展示（绿色底）
- `.location-label-text` — 定位文本（绿色）
- `.location-clear` — 清除按钮
- `.location-tags` / `.location-tag` — 地点标签网格
- `.location-tag.selected` — 选中态（蓝色填充）
- `.optional` — "可选"灰色文字
- `.preview-location` / `.preview-location-text` — 预览页位置展示
- `.preview-location-edit` — 预览页位置编辑入口

#### 4. `cloudfunctions/publishPost/index.js` — **更新**

**新增字段支持：**
- `location` — 位置名称/描述（字符串）
- `latitude` — GPS 纬度（数字）
- `longitude` — GPS 经度（数字）

写入数据库 `posts` 集合后，帖子详情页已可直接展示位置信息（原有 UI 支持）。

### 功能流程

```
页面加载 → _autoGetLocation() 静默获取 GPS
    ↓
用户点击「获取 GPS 定位」→ 授权引导 → 定位成功填充经纬度
    或
用户点击地点标签 → selectedLocation 填充标签名
    ↓
预览模式展示位置信息（标签名或经纬度）
    ↓
发布时注入 location/latitude/longitude 到云函数
```

## 五、技术说明

### 关于 GPS 权限

微信小程序使用 `wx.getLocation` 需要用户授权，且需在 `project.config.json` 中配置 permission（已在现有项目配置中具备）。生产环境需在小程序后台 → 开发管理 → 开发设置 → 地理位置 申请开通。

### 关于隐私政策

使用位置功能前需在**小程序后台 → 设置 → 隐私设置**中声明位置权限用途，否则审核可能被拒。

### 关于经纬度存储

`latitude`/`longitude` 以数字类型存入数据库，可用于后续「猫咪地图」功能的坐标展示（P2-M13）。
