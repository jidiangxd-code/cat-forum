## 二、本次开发执行 — P1-4 子评论/回复功能 ✅

### 需求分析

需求文档（P1-4）要求：
- 评论为单层嵌套结构，点击"回复"按钮弹出回复输入框并@被回复用户
- 子评论最多两层（主评论 → 回复，不允许回复的回复再被回复）
- 评论后向帖子作者和被子评论用户发送通知

### 实现方案

**前端改动（detail 页面）：**

1. **detail.js — 数据层 + 逻辑层**
   - `data` 新增 5 个字段：`isReply`、`replyToCommentId`、`replyToUserId`、`replyToUserName`、`replyPlaceholder`
   - `loadComments()` 重构 — 评论数据按 `parentId` 分组，将子评论聚合到对应主评论的 `replies` 数组中
   - `onReplyComment()` — 点击回复按钮，切换回复模式，记录被回复用户信息（不能回复自己）
   - `onCancelReply()` — 取消回复模式，恢复普通输入
   - `submitComment()` 重构 — 提交时携带 `parentId`、`replyToUserId`、`replyToUserName` 字段（仅回复模式）
   - 新增 `onReplyAvatarError()` — 子评论头像加载失败兜底

2. **detail.wxml — 页面结构**
   - 评论列表用 `wx:if="{{!item.parentId}}"` 筛选主评论
   - 每条主评论下增加 `.reply-list` 子评论区域（灰色背景 + 左侧橙色竖线）
   - 子评论显示「回复者头像 + 名字 + [回复] + @被回复者 + 内容 + 时间」
   - 输入框上方新增「回复提示条」（显示 @被回复用户 + 取消按钮），回复模式下自动聚焦

3. **detail.wxss — 样式**
   - `.comment-body` — 包裹主评论内容区（flex 布局）
   - `.comment-top` — 作者名 + 时间并排
   - `.reply-btn` — 橙色小按钮（回复操作）
   - `.reply-list` — 灰色背景子评论区，左侧 3rpx 橙色竖线
   - `.reply-header` — 子评论头部（头像 + 作者 + 回复 + @人 + 时间）
   - `.reply-text` — 子评论正文（缩进 48rpx）
   - `.reply-hint-bar` — 底部输入框上方提示条（回复模式显示）

4. **api.js — 接口层**
   - `addComment()` 新增 3 个可选参数：`parentId`、`replyToUserId`、`replyToUserName`
   - 有 `parentId` 时写入 `comments` 集合对应字段

## 三、通知发送链路确认

经检查，`addComment` 云函数目前仅写入 `comments` 集合，**未调用** `sendNotification` 云函数。

评论通知需在以下时机触发：
1. 用户评论帖子 → 通知帖子作者
2. 用户回复某条评论 → 通知被回复用户 + 帖子作者

建议后续在 `detail.js` 的 `submitComment()` 成功后补充调用 `api.sendNotification()`（目前 `api.js` 尚未封装该方法），或改造 `addComment` 云函数内联发送通知。
