## 二、本次开发执行 — P1-2 消息通知中心 ✅

### 现状分析

通过检查发现，消息通知模块的基础设施（云函数 + 数据库集合 + WXML）**已经存在**，但核心 JS 逻辑文件缺失，导致页面完全不可用。

**已具备：**

- ✅ `cloudfunctions/markNotificationRead/index.js` — 标记已读（完整）
- ✅ `cloudfunctions/sendNotification/index.js` — 发送通知（完整）
- ✅ `pages/notifications/notifications.wxml` — 页面结构（完整）
- ✅ `pages/notifications/notifications.json` — 页面配置（完整）
- ❌ `pages/notifications/notifications.js` — **核心逻辑（缺失）**
- ❌ `pages/notifications/notifications.wxss` — 样式（缺失）
- ❌ 个人中心未接入通知入口和未读角标

### 开发内容

#### 1. 新建 `notifications.js`（核心逻辑）

实现以下功能：
- `loadNotifications()` — 调用云函数加载通知列表，支持分页和 Tab 筛选（全部/获赞/评论/关注）
- `loadUnreadCount()` — 获取未读数，并同步设置 TabBar 角标（第4个Tab，即"我的"）
- `switchTab()` — Tab 切换，刷新对应类型通知
- `onNotificationTap()` — 点击单条通知，标记已读 + 跳转（帖子/猫咪详情页）
- `markAllRead()` — 全部已读，同时更新 TabBar 角标
- `getNotifText()` — 通知类型 → 展示文本映射
- `onPullDownRefresh()` + `onReachBottom()` — 下拉刷新 + 上拉加载更多
- `onImageError()` — 图片加载失败兜底

#### 2. 新建 `notifications.wxss`（样式）

完整样式覆盖：
- Tab 栏（粘性定位 + 选中态下划线 + 未读数角标红点）
- 通知列表（头像区 + 类型图标标签 + 内容区 + 封面图）
- 未读状态高亮（左侧橙色竖条 + 背景色）
- 空状态 + 加载中 + 加载更多
- 底部固定「全部已读」按钮（悬浮胶囊样式）

#### 3. 个人中心接入通知入口

**profile.wxml：** 在"我的内容"菜单前新增"消息"区块
- 🔔 图标 + "消息通知"文字入口
- 右侧显示未读数红点角标（wx:if="{{unreadCount > 0}}"）
- 仅登录用户可见

**profile.js：**
- `data` 新增 `unreadCount: 0`
- `onShow()` 新增 `this.loadUnreadCount()` 调用
- 新增 `goToNotifications()` → `wx.navigateTo('/pages/notifications/notifications')`
- 新增 `loadUnreadCount()` 方法 — 调用 getNotifications 云函数获取未读数

**profile.wxss：** 新增 `.menu-right` + `.badge` 样式（红色未读角标）

#### 4. app.json 注册通知页

- 在 `pages/detail/detail` 后插入 `pages/notifications/notifications`
